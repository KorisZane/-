# Web网站架构

## 套用模板

*csdn/cnblog/github/建站系统等*

中小型应用，测试时测试网站使用的源码或者利用已知的n day

一般以模板套用，基本模板无漏洞，大部分采用测试用户管理权限为主

## 前后端分离

前后端分离就是前端页面和后端管理不在一个服务器上，前端服务器只做一个渲染和像后端传送数据，一个后端可以控制多个前端页面服务器

示例网站 https://www.rxthink.cn/

思路文章 https://mp.weixin.qq.com/s/HtLU_EBXWcbq-lt10oPYwA

安全测试以前端JS(Vue,Nodejs等）安全问题，主要以API接口测试，前端漏洞为主，后端隐蔽难度加大

大部分小程序或者app使用前后端分离架构

判断方法就是访问不同页面时页面的URL不变

## 集成软件包

*宝塔 phpstudy xmap等*

主要以软件的防护策略为主 *例如给宝塔搭的网站上传后门，无法执行命令，只能查看网站目录的文件，给phpstudy搭的网站上传后门可以执行命令，查看全部文件*

## 自主环境镜像

云镜像打包，测试时可以执行权限，出去高权限目录（root目录等），文件管理基本都能看

## 容器拉取镜像

内置容器技术，虚拟化磁盘空间，需要进行docker逃逸

## 纯静态页面

只使用html css javascript 无安全漏洞
测试时找其他资产 域名 客户端等
